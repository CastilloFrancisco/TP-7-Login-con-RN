La realización de este trabajo práctico resultó ser una experiencia sencilla y directa desde el punto de vista del desarrollo frontend. El objetivo consistió en construir una aplicación de inicio de sesión utilizando React Native y Expo, implementando una interfaz básica con campos para correo electrónico y contraseña, un botón de acceso y mensajes de validación según las credenciales ingresadas por el usuario.

Desde la perspectiva técnica, el desarrollo no presentó grandes desafíos relacionados con la lógica de programación. La utilización de componentes nativos como `SafeAreaView`, `View`, `Text`, `TextInput`, `Image` y `TouchableOpacity` permitió estructurar la interfaz de manera clara y organizada. Asimismo, el uso del hook `useState` facilitó la gestión del estado de los campos de entrada y de los mensajes mostrados al usuario, aplicando conceptos fundamentales de React de forma práctica y efectiva.

En cuanto al diseño visual, se trabajó con hojas de estilo mediante `StyleSheet`, definiendo la disposición de los elementos, los colores corporativos de la aplicación y una distribución centrada que favorece la experiencia de usuario. La inclusión de una imagen y de un encabezado fijo contribuyó a generar una interfaz más completa y agradable visualmente, manteniendo al mismo tiempo una estructura simple y funcional.

La única dificultad relevante surgió durante la ejecución del proyecto. Inicialmente, la aplicación no lograba iniciarse correctamente utilizando el comando habitual con la opción `-tunnel`. Luego de realizar algunas pruebas y revisar posibles causas relacionadas con la configuración del entorno de desarrollo, se encontró una solución sencilla ejecutando el proyecto mediante el comando `npx expo start -c`. La opción `-c` permitió limpiar la caché de Expo y regenerar los archivos temporales necesarios para la ejecución, resolviendo el inconveniente sin necesidad de realizar modificaciones en el código fuente.

Esta situación puso en evidencia la importancia de comprender no solo el desarrollo de la aplicación en sí, sino también las herramientas que intervienen en el proceso de compilación y ejecución. En muchos casos, los problemas no se encuentran en la lógica implementada, sino en aspectos relacionados con la configuración del entorno, las dependencias o los archivos de caché generados por los frameworks utilizados.

En conclusión, este trabajo práctico permitió reforzar conocimientos básicos de React Native, manejo de estados, creación de interfaces móviles y uso de Expo como plataforma de desarrollo. Si bien la complejidad técnica del ejercicio fue baja, la experiencia resultó útil para afianzar conceptos fundamentales y para adquirir una mejor comprensión de los problemas que pueden surgir durante la puesta en marcha de una aplicación móvil.
